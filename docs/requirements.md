# 智能招聘系统需求文档

## 1. 系统概述

该系统是为HR公司量身定制的智能招聘平台，旨在通过自动化和智能化流程优化招聘环节，提高招聘效率，减少人工操作，并通过AI技术提高招聘精度。系统涵盖简历筛选、招聘需求管理、面试安排、面试反馈和入职管理等核心功能。

## 1.1 AI功能要求

所有系统中的推理过程必须使用GPT-4模型，包括：
1. 简历解析和内容提取
2. 人才画像生成
3. 技能标签生成
4. 招聘需求匹配
5. 面试问题生成

## 2. 系统功能模块

### 2.1 简历管理模块

功能描述：
1. 简历上传：
   - 支持格式：PDF、Word、Excel，最大支持100MB
   - 批量上传：支持选择多个文件一次性上传（最多10个）
   - UI：上传区域包括文件选择框、上传按钮、进度条
   - 错误处理：显示中文错误提示，包括文件格式、大小限制等

2. 简历解析：
   - OCR识别：上传后简历通过阿里云OCR进行文本提取
   - 标签化：系统使用GPT-4自动生成简历中的技能标签（如技术栈、经验年限等）
   - 生成画像：使用GPT-4生成一段概括候选人能力的简短文字描述（人才画像）
   - UI：左侧展示原始简历，右侧展示解析后的简历内容与标签

3. 简历确认与存储：
   - 确认信息：HR确认简历中的解析数据，修改错误信息后，点击"确认"
   - 文件夹管理：简历可以存储到用户自定义的虚拟文件夹中，支持文件夹的创建、查看和删除
   - 简历列表：支持按上传时间排序，支持搜索候选人姓名

4. 简历标签管理：
   - 自动标签：系统使用GPT-4自动为简历生成标签（如技能、经验年限、学历等）
   - 手动编辑：HR可以添加、删除、修改标签
   - 标签分类：支持按技能、经验、教育等类别管理标签

### 2.2 招聘需求管理模块

功能描述：
1. 创建招聘需求：
   - 招聘需求表单：包括职位名称、部门、岗位职责、技能要求、学历要求、工作年限、薪资范围、工作地点等
   - 自动提取功能：HR上传招聘需求文档时，系统使用GPT-4自动提取关键内容，填充至表单中
   - 薪资范围：支持设置最低和最高薪资，自动验证范围合理性
   - 地点选择：支持省市区三级联动选择工作地点

2. 招聘需求匹配：
   - 智能匹配：系统使用GPT-4对简历和职位进行多维度匹配分析
   - 匹配维度：包括技能匹配度、经验匹配度、教育匹配度、职责匹配度
   - 匹配评分：每个维度0-100分，并提供详细的评分理由
   - 双向匹配：支持职位找简历和简历找职位两种匹配方式

3. 招聘需求状态与流程：
   - 状态管理：职位状态包括待发布、招聘中、已暂停、已结束
   - 进度跟踪：HR可以查看每个职位的应聘者列表和匹配分数
   - 批量操作：支持批量更新职位状态

### 2.3 面试管理模块

功能描述：
1. 面试预约：
   - 选择候选人：从候选人列表中选择进行面试的候选人
   - 选择面试官：面试官列表展示所有面试官，支持选择一个或多个面试官
   - 选择面试时间：系统自动同步面试官和候选人的日程，避免冲突

2. 面试问题生成：
   - 自动生成：系统基于职位要求与候选人背景生成面试问题
   - 手动编辑：HR可以编辑生成的问题，或者选择不同的面试问题

3. 面试进度与反馈：
   - 面试反馈：面试后，HR填写候选人的面试评估，评价是否合适
   - 面试状态：候选人面试完成后，状态更新为"completed"

### 2.4 入职管理模块

功能描述：
1. 入职登记：
   - 录入信息：面试通过后，HR录入候选人的入职信息（如岗位、入职日期、薪资等）
   - 入职确认：生成入职确认书，发送给候选人确认
   - 资料收集：收集候选人的入职所需资料

2. 入职流程跟踪：
   - 状态管理：跟踪入职流程的各个环节（如资料收集、入职确认等）
   - 提醒功能：自动提醒HR和候选人待办事项
   - 文档管理：管理入职相关文档

## 3. 技术要求

1. 后端开发：
   - 使用Python语言
   - 使用MySQL数据库
   - 使用阿里云OCR服务
   - 使用GPT-4进行AI推理

2. 前端开发：
   - 使用Vue.js框架
   - 使用Element Plus组件库
   - 支持响应式设计

3. 测试要求：
   - 在测试环境中进行所有测试
   - 使用真实的测试用例，不使用模拟数据
   - 在前端完整执行1-2个端到端流程测试
   - 测试所有可能受影响的功能模块
   - 主动监控并收集前端日志和控制台(F12)信息进行调试

4. 开发过程要求：
   - 需要随时对照需求文档进行开发
   - 保持对当前计划的清晰认知
   - 定期确认已完成的进度
   - 及时汇报开发状态和下一步计划
